1. 로그인 페이지
- 우리 웹사이트의 시작페이지
- 로그인 안하면 게시글 목록 및 게시글 상세 조회 불가
- 메인 배경: 브라우저 가운데에 위치
- 우상단 언어선택: 셀렉박스만 구현(한/영만). 실제로 변경은 안된다.
- 네이버 로고 구현
- 일회용 번호 / QR코드는 작동 안해야 하는데, 만들지도 않아도 된다
- 아이디 누르면 테두리 녹색으로 변함, 커서 깜빡임, 입력값 모두 지우는 X버튼 구현
- 비밀번호만 비어있을 때 로그인 누르면 경고문구 뜨고, 커서와 포커스가 비밀번호 칸으로 넘어감
- IP보안 필요없음
- 로그인 상태 유지는 선택적으로(쿠키 관련). 모양은 만들기
- 비번찾기 눌렀을 때 페이지도 만들기. 내 아이디 넣고 다음 누르면 페이지 넘어가는거까지 하는데, 그 마지막 페이지는 대충 만들어도 됨
- 아이디 찾기는 네이버 아이디만 되면 됨. 다음 페이지는 네이버아이디/로그인전용 아이디만 있는데 작동은 네이버아이디만 됨.
	이름, 휴대전화 아이디 찾아져야 함. 밑에는 아예 필요없음 
- 하단 광고, footer는 작동은 안되지만 생긴건 똑같아야 함(이미지도 네이버 이미지 써야 함)
- 로그인 화면에서 아이디, 비번 입력하고 상단 Naver로고 누르면, 같은 페이지라도 입력내용은 없어져야 함
- 로그인 페이지에서 로그인하면 게시판목록으로 감

2. 회원가입
- 회원가입 하기도 언어는 안바뀌어도 됨
- 회원가입 약관 내용은 적당히 스크롤 생길 정도로만
- 체크 안하면 "다음" 작동 안됨
- 필수를 모두 체크하면 다음 활성화
- 어린이용은 작동 안해도 됨
- 단체, 비즈니스도 작동 안해도 됨
- 필수정보 없으면 오류문구, 빨간 테두리 생김
- 비밀번호 안전성 관련은 사용불가/가능 정도로만 하기
- 비밀번호 눈알(*으로 표시 - 내용 표시 전환)도 해보기(인풋 패스워드 안해도 됨)
- 통신사는 없던 있든 마음대로
- 010은 여유 좀 봐서 엄격하게 하든 안하든 => 0810 별도로 다시 정리: 전화번호 형식 유효성 검사는 해야 함.
- 필수 체크까지는 하는데, 밑에 열리는거는 안함
- '실명인증된 아이디로 가입'은 안해도 됨
- 인증요청 버튼누르면 회원가입 완료되고 로그인으로 돌아간다
	=> 0810 별도로 다시 정리:  인증번호 가는 것도 있음. 문자가 오는 것은 경고창/팝업 등으로 융통성있게 대체.
	인증번호 ctrl+c 또는 베껴쓰거나, 팝업창을 끄면 자동 입력되도록 등등 융통성있게. 생성된 인증번호와 입력한 인증번호가 일치해야 진행됨.

3. 게시판
- 게시판 목록도 로그인/회원가입과 배경 동일하고 가운데정렬
- 게시판 목록에도 상단 좌측에  NAVER로고 뜨고, 우측에는 프로필사진(선택), 이름, 계정 표시, 로그아웃버튼 구현

- 게시판 상세는 배경 색 다름
- 이름, 이메일은 고정
- 제목, 내용은 비어있음
- 비밀번호도 구현
- 하단에 목록보기 버튼=우상단 글목록 버튼
- 글 쓸 때와 글 조회할 때의 전반적인 ui는 동일하나 버튼 안에 문구 등은 적당히 바꾸자.


- '카테고리명'은 '제목'으로, 'Description'은 '내용' 으로 등 게시판 상단 이름도 팀원들과 상의해 적당히 바꿔준다.
- 게시판 상단 순서도 적당히 변경 가능하다.
- 검색 기능은 두 가지로 가능함(제목, 내용, 사용자명 등 중에 2개 선택해서 구현)
- 로그아웃 버튼 누르면  -> 실제 네이버 로그인 하면 나오는 내 정보 표시칸 같은 것에 yes/ no 선택버튼 구현하여
	바로 로그아웃 되는 것이 아니라 로그아웃 할지, 안할지 선택할 수 있도록 구현한다(yes 누르면 로그인 화면으로 이동)
